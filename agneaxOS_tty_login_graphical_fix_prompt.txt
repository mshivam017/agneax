WHAT THE SCREENSHOT MEANS

The ISO is booting successfully into Debian/Linux.
This is not a kernel panic and not a dead ISO.

The problem is after boot:

systemd -> LightDM -> Agneax Wayland session -> Weston -> PySide6 desktop

That graphical chain is not reaching the desktop, so the system leaves you at tty1 console login.

Current login from the ISO should be:

Username: agneax
Password: agneax

If you selected "Agneax OS Debug Console" in GRUB, this tty1 screen is expected.
If you selected "Agneax OS Live (Standard Mode)" and still got tty1, LightDM or the Weston session failed.

LIKELY ROOT CAUSES

1. LightDM was installed without a greeter
- build_iso.sh uses --no-install-recommends.
- lightdm can be installed without lightdm-gtk-greeter.
- If no greeter/session startup path is available, graphical login may fail.

2. Default target may not be graphical.target
- The OS can boot to multi-user console if graphical target is not set correctly.

3. Agneax Wayland session may not launch correctly
- /usr/bin/agneax-session-start starts Weston.
- Weston needs a valid runtime directory and DBus session.
- Without XDG_RUNTIME_DIR/DBus, Weston can fail or exit.

4. VirtualBox graphics can fail with Weston DRM
- Weston --backend=drm is sensitive to VM graphics settings.
- Use VMSVGA, 128 MB video memory, and test 3D acceleration ON/OFF.

5. If desktop crashes after Weston starts
- Logs should be checked:
  - /tmp/weston.log
  - /tmp/agneax-desktop.log

VIRTUALBOX SETTINGS TO USE

Recommended:
- Type: Linux
- Version: Debian 12 64-bit
- RAM: 4096 MB minimum
- CPU: 2 cores minimum
- Graphics Controller: VMSVGA
- Video Memory: 128 MB
- Enable 3D Acceleration: try ON first

- Boot order: Optical first

Try starting the graphical service manually:

sudo systemctl restart lightdm

If LightDM starts but desktop is blank, test manually:

export QT_QPA_PLATFORM=wayland
export QT_WAYLAND_SHELL_INTEGRATION=kiosk-shell
cd /opt/agneax/desktop
python3 main.py

If PySide6 fails:

python3 -c "import PySide6; print(PySide6.__version__)"
python3 -c "from PySide6.QtQml import QQmlApplicationEngine; print('QML OK')"

IF YOU WANT A TEMPORARY MANUAL START FROM TTY

After login:

export XDG_RUNTIME_DIR=/run/user/$(id -u)
mkdir -p "$XDG_RUNTIME_DIR"
chmod 700 "$XDG_RUNTIME_DIR"
dbus-run-session -- weston --backend=drm --shell=kiosk-shell.so --continue-without-input --log=/tmp/weston.log --startup-cmd="/opt/agneax/desktop/run.sh"

If this works, the problem is LightDM startup.
If this fails, read /tmp/weston.log.
